Cahier des charges rédactionnel Protokin

1. Finalité des documents

Les documents produits dans le cadre de Protokin ont pour fonction :

- de décrire des régimes descriptifs ;
- d'expliciter leurs conditions de validité ;
- de cartographier leurs tensions ;
- d'analyser leurs transitions ;
- de formaliser des mécanismes de stabilisation.

Ils ne visent jamais à produire une théorie totale du réel.

---

2. Principe fondamental

Toute affirmation doit être formulée comme une proposition portant sur un régime descriptif ou un cadre d'observation.

Éviter les formulations :

- « le réel est »
- « l'homme est »
- « la conscience est »
- « la société est »

Préférer :

- « dans ce régime descriptif »
- « ce cadre sélectionne »
- « cette description stabilise »
- « cet invariant apparaît lorsque »

---

3. Interdictions conceptuelles

Ne jamais écrire :

- qu'un régime est plus vrai qu'un autre ;
- qu'un régime remplace un régime faux ;
- qu'une transition constitue un progrès nécessaire ;
- qu'un niveau émerge mécaniquement d'un autre ;
- qu'une tension constitue une erreur du réel.

Ne jamais introduire :

- de téléologie ;
- de hiérarchie ontologique implicite ;
- de réductionnisme non justifié ;
- d'analogies littéraires utilisées comme démonstrations.

---

4. Style rédactionnel

Privilégier :

- un style impersonnel ;
- des formulations descriptives ;
- des phrases courtes ;
- un vocabulaire technique stable.

Limiter :

- les effets rhétoriques ;
- les métaphores ;
- les formulations militantes ;
- les jugements de valeur.

Le texte doit pouvoir être lu comme un document d'architecture conceptuelle.

---

5. Structure recommandée

Chaque document doit comporter :

Identification

- Nom
- Statut
- Domaine
- Type

Définition

Définition courte et rigoureuse.

Fonction

Rôle joué dans l'architecture.

Conditions de validité

Conditions sous lesquelles la description reste cohérente.

Limites

Situations où le régime cesse de stabiliser les invariants concernés.

Tensions

Relations avec les autres régimes.

Transitions associées

Transitions possibles vers d'autres cadres descriptifs.

---

6. Vocabulaire privilégié

Utiliser préférentiellement :

- stabilisation
- invariant
- couplage
- contrainte
- audit
- régime
- transition
- tension
- domaine de validité
- reconfiguration
- sélection
- maintenance

Éviter lorsque possible :

- vérité absolue
- essence
- nature profonde
- révélation
- dépassement final
- stade ultime
- accomplissement

---

7. Utilisation des sources

Toute source mobilisée doit être :

- reconstruite dans le vocabulaire de Protokin ;
- explicitement localisée dans un régime descriptif ;
- analysée pour ses conditions de validité.

Les auteurs ne doivent jamais être traités comme des autorités définitives.

Ils sont considérés comme des producteurs de cadres descriptifs analysables.

---

8. Critère de qualité

Un texte Protokin est considéré comme réussi lorsqu'il permet :

- d'identifier clairement le régime étudié ;
- d'expliciter ses invariants ;
- de décrire ses limites ;
- de comprendre ses tensions avec d'autres régimes ;
- de situer les transitions possibles.

Le document doit augmenter la lisibilité de l'architecture sans produire de confusion entre description et ontologie.
