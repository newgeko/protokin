Prompt image générateur canonique Protokin (v1.0)

1. Fonction

Ce prompt sert à générer des images conformes à l’architecture Protokin à partir de n’importe quel concept, texte ou idée.

Il opère une transformation automatique :

«concept → régime(s) descriptif(s) → structure visuelle → image abstraite stable»

---

2. Instruction principale

À partir de l’entrée utilisateur, identifier :

1. les régimes descriptifs impliqués (Proto / Kin / mixte)
2. les relations entre ces régimes (stabilité, tension, transition)
3. les invariants structurels du concept
4. la forme visuelle optimale pour représenter ces relations

Puis générer une image uniquement sous forme de structure abstraite, sans narration ni symbolisation littérale.

---

3. Décomposition obligatoire

Toute génération d’image doit respecter les étapes suivantes :

3.1 Analyse conceptuelle

Identifier si le contenu relève principalement de :

- Proto : champs continus, causalité implicite, gradients, stabilisation physique/cognitive
- Kin : structures relationnelles, normes, réseaux, interactions explicites
- Tension : incompatibilités locales entre régimes
- Transition : zones hybrides, reconfigurations, passages

---

3.2 Traduction structurelle

Convertir le contenu en structure visuelle :

- Proto → champs, gradients, continuités
- Kin → nœuds, graphes, relations explicites
- Tension → forces opposées, frictions, ruptures
- Transition → zones intermédiaires, superpositions, flous structurés

---

3.3 Composition globale

Construire une image respectant :

- hiérarchie non narrative
- organisation modulaire
- lisibilité des régimes
- absence de centre symbolique dominant

---

4. Contraintes stylistiques obligatoires

L’image doit respecter :

- abstraction structurelle stricte
- absence de personnages, scènes ou narration
- absence de symboles culturels explicites
- esthétique minimaliste ou schématique
- palette désaturée (gris, noir, blanc + une couleur fonctionnelle max)

---

5. Règles de cohérence Protokin

Interdictions absolues :

- toute illustration métaphorique (ex : “cerveau = réseau lumineux”)
- toute personnification des concepts
- toute narration implicite
- toute hiérarchie ontologique visuelle (pas de “niveau supérieur” visuel)

---

6. Format de sortie attendu (prompt final image)

Le système doit produire un prompt image structuré ainsi :

A. Description structurelle

- type de système (Proto / Kin / mixte / tension / transition)
- nature des relations (causal, normatif, hybride)
- dynamique globale (stabilité / friction / reconfiguration)

B. Composition visuelle

- organisation spatiale des éléments
- type de structure (champ, réseau, gradient, hybridation)
- zones de tension ou de transition

C. Style

- abstraction analytique
- minimalisme structurel
- lisibilité élevée
- absence de narration

---

7. Critère de validation

Une image est conforme si :

- elle est lisible comme une structure et non comme une scène
- les relations priment sur les objets
- plusieurs interprétations narratives sont impossibles ou inutiles
- les régimes (Proto / Kin) sont différenciables visuellement
- les tensions et transitions sont perceptibles sans explication

---

8. Formule synthétique opératoire

«Transformer tout concept en configuration de régimes descriptifs visualisés comme structures abstraites, où les relations, tensions et transitions remplacent toute narration.»
