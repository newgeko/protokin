# 🧭 PAR — Protokin Audit des Régimes

## 🧩 1. Définition générale

PAR (*Protokin Audit des Régimes*) est un module d’analyse descriptive interne à l’architecture Protokin.

Il a pour fonction d’expliciter les conditions de constitution, de stabilisation et de compatibilité des régimes de description.

PAR ne produit pas de nouveaux régimes.

Il analyse les conditions sous lesquelles des régimes existent comme cadres de contraintes descriptives.

---

## ⚙️ 2. Statut du module

PAR n’est pas :

- un opérateur causal  
- un mécanisme de transformation  
- un niveau ontologique  
- une instance de décision  
- un système d’évaluation externe  

👉 PAR est un **niveau de modélisation des conditions de constitution des régimes**.

---

## 🧠 3. Objet d’analyse

PAR porte sur trois dimensions fondamentales :

### 🔹 3.1 Constitution des régimes

PAR analyse :

- quelles contraintes définissent un régime ;
- quelles règles stabilisent ses critères de validité ;
- comment un espace de description devient cohérent localement.

---

### 🔹 3.2 Stabilisation

PAR examine :

- les conditions de maintien d’un régime ;
- les mécanismes d’ajustement interne ;
- les formes de cohérence locale.

---

### 🔹 3.3 Compatibilité inter-régimes

PAR identifie :

- les zones de recouvrement partiel ;
- les incompatibilités structurelles ;
- les conditions de co-présence sans fusion.

---

## 🧩 4. Structure logique des régimes (vue PAR)

Dans une perspective PAR, un régime peut être décrit comme :

- un ensemble de contraintes de recevabilité ;
- un ensemble de règles d’inférence locales ;
- un espace de distinctions pertinentes ;
- un système de filtrage de ce qui compte comme description valide.

👉 PAR ne décrit pas les contenus, mais les **conditions de leur possibilité locale**.

---

## ⚠️ 5. Limites de l’analyse PAR

PAR ne permet pas :

- de hiérarchiser les régimes ;
- de déterminer leur vérité ;
- de les unifier ;
- de les réduire à un cadre unique ;
- de les valider ou invalider globalement.

👉 PAR décrit des **conditions de constitution**, pas des valeurs de vérité.

---

## 🔄 6. Relations avec les autres modules

### ⚙️ Kernel

- PAR : décrit les conditions de constitution des régimes  
- Kernel : régule les relations entre régimes constitués  

---

### ⚡ OVM

- PAR : identifie les structures de régimes  
- OVM : détecte les transpositions illégitimes entre régimes  

---

### 🚧 DELTA

- PAR : travaille dans les limites des cadres analysables  
- DELTA : signale l’impossibilité de stabilisation de ces cadres  

---

### 🔁 PASS

- PAR : structure théorique  
- PASS : mise en œuvre procédurale de l’analyse PAR sur un corpus  

---

## 🧠 7. Principe fondamental

> Un régime n’est pas donné : il est **constitué par un ensemble de contraintes de stabilisation descriptive**.

PAR explicite ces contraintes sans les produire.

---

## 🧭 8. Fonction épistémologique

PAR permet de :

- rendre explicites les conditions implicites de description ;
- analyser la pluralité des cadres sans les réduire ;
- éviter les glissements entre description et ontologie ;
- stabiliser la cartographie des régimes sans hiérarchie.

---

## 🔒 9. Garde-fous

PAR n’introduit jamais :

- de métaphysique des régimes ;
- de causalité entre régimes ;
- de hiérarchie normative ;
- de réduction inter-régimes ;
- de validation externe des descriptions.

---

## 🧭 Conclusion

PAR est le module qui rend possible une lecture structurée des régimes de description sans jamais les transformer en objets ontologiques.

Il explicite les conditions de leur constitution, sans intervenir sur leur contenu ni sur leur vérité.
